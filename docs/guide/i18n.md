# Internationalization (i18n)

CheckAI supports 8 languages out of the box with full coverage of all user-facing strings in both the backend and the web frontend.

## Supported Languages

| Code    | Language             |
| ------- | -------------------- |
| `en`    | English (default)    |
| `de`    | German               |
| `fr`    | French               |
| `es`    | Spanish              |
| `zh-CN` | Chinese (Simplified) |
| `ja`    | Japanese             |
| `pt`    | Portuguese           |
| `ru`    | Russian              |

## Backend (Rust)

The backend uses the [`rust-i18n`](https://crates.io/crates/rust-i18n) crate with YAML locale files.

### Locale Files

Translations are stored in `locales/*.yml`:

```bash
locales/
├── en.yml
├── de.yml
├── fr.yml
├── es.yml
├── zh-CN.yml
├── ja.yml
├── pt.yml
└── ru.yml
```

### Usage in Code

```rust
use rust_i18n::t;

// Simple key lookup
let msg = t!("game.created");

// With interpolation
let msg = t!("game.move_made", from = "e2", to = "e4");
```

### Locale Resolution

1. `--lang` CLI flag
2. `CHECKAI_LANG` environment variable
3. System locale (auto-detected via `sys-locale`)
4. Fallback: English

### Per-Request Locale (API)

API requests can specify a locale:

```bash
# Query parameter
GET /api/games?lang=de

# Accept-Language header
GET /api/games
Accept-Language: de
```

## Frontend (Web UI)

The web frontend has its own translation module at `web/js/i18n.js` with ~120 keys per language.

### Features

- **Auto-detection** from browser `navigator.language`
- **Language selector** dropdown in the header
- **Persistence** via `localStorage`
- **`data-i18n` attributes** for declarative HTML translations
- **`t()` function** for programmatic access

### Adding a Translation

To add a new language to the frontend:

1. Add the language code and translations in `web/js/i18n.js`
2. Add a YAML locale file in `locales/`
3. The backend will automatically pick it up
